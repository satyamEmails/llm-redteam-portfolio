TITLE:
Triple Vulnerability Chain in Enterprise RAG Systems:
Document Poisoning + Cross-User Leakage +
Indirect Injection via Retrieved Content

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DEPLOYMENT CONTEXT:
Enterprise RAG system for BFSI sector.
AI reads internal documents to answer
customer and employee queries.
60% of enterprise AI deployments use RAG.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

THREE ATTACKS DEMONSTRATED:

ATTACK 1: Document Poisoning
→ Attacker uploads malicious document
→ All users asking related questions
   receive compromised responses
→ Used for: Phishing, misinformation,
   reputation damage
→ OWASP: LLM03 — Data Poisoning
→ Severity: CRITICAL

ATTACK 2: Cross-User Data Leakage  
→ Crafted queries retrieve other users' data
→ Privacy violation at scale
→ Used for: Intelligence gathering,
   identity theft, regulatory violation
→ OWASP: LLM06 — Sensitive Info Disclosure
→ Severity: CRITICAL

ATTACK 3: Indirect Injection via RAG
→ Retrieved documents contain hidden
   AI instructions
→ AI follows attacker instructions
   thinking they came from trusted docs
→ In agentic systems: triggers real actions
→ OWASP: LLM01 — Prompt Injection
→ Severity: CRITICAL + AGENTIC AMPLIFIER

RECOMMENDED FIXES:

1. Document Poisoning:
   → Strict document upload authorization
   → Content scanning before ingestion
   → Human review for sensitive documents

2. Cross-User Leakage:
   → User-level access control in vector DB
   → Query results filtered by user permissions
   → Separate namespaces per user/department

3. Indirect Injection:
   → Separate document content from instructions
   → Never treat retrieved content as trusted
   → Output filtering for sensitive actions

4. Agentic Protection:
   → Human approval for high-risk actions
   → Least privilege for agent permissions
   → Action audit logs with anomaly detection
